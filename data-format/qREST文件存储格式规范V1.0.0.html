qREST_DATA 结构监测数据格式说明书

0.概述

qREST_DATA 是一种专门为建筑结构轻量化地震监测（Light-weight S²HM）设计的数据格式规范，旨在提供一个统一、标准化的框架来存储和交换监测数据。该数据格式包含三个主要部分：文件头（File Header）、元数据（Metadata）以及时序数据（Time Series Data）或称数据包（Body）。

文件头部分采用固定长度的二进制格式，包含了文件标识、元数据长度和数据体长度等关键信息，用于快速识别文件格式和定位数据内容。

元数据基于 JSON 格式设计，旨在提供标准化的元数据（Metadata）存储规范。元数据采用模块化的设计，除了数据说明、版本号和单位约定外，
包含三大核心模块：建筑结构信息、传感器布设信息以及时序数据属性。得益于 JSON的灵活性和可读性，元数据能够清晰地描述监测项目的基本信息、传感器配置和数据属性，并允许用户根据实际需求进行扩展和定制。只要本项目中定义的核心字段保持不变，用户可以在元数据中添加任意数量的自定义字段，以满足特定项目的需求。

数据包使用二进制进行编码，以高效存储和传输大规模的时序数据。为了保证数据的正确解析和使用，时序数据分为数据包头和数据包体两部分。包头包含了当前数据块的时间属性和采样参数。数据包则包含了每个通道的时序数据，按照预定义的顺序进行存储，以便程序能够正确读取和处理。

1.物理与工程约定

在解析本数据文件时，需严格遵循以下工程约定：

  • 地理正北角 (North Angle)：
    以结构局部坐标系的 Y 轴正方向与地理正北方向之间的夹角定义，单位为度（°）。例如：如果 Y 轴正向与地理正北方向重合，则 North Angle 为 0°；如果 Y 轴正向指向东，则 North Angle 为 270°。
  • 结构局部坐标系 (Local Coordinate System)：
    • 原点 (Origin)：以建筑底平面（正负零标高处）的几何中心为三维坐标系的原点 [0.0, 0.0, 0.0]。
    • 轴向 (Axes)：遵循右手笛卡尔坐标系。通常 X 轴与建筑主轴线（如长边）平行，Y 轴与次轴线（如短边）平行，Z 轴垂直向上。
  • 方向角定义 (Azimuth / Direction)：
    采用极坐标角度定义方式：以结构局部坐标系的 X 轴正方向为 0°，在 XY 平面内逆时针旋转至 360°。例如：Y 轴正向为 90°，X 轴负向为 180°。

2.文件头详细说明

文件头固定为 16字节，采用 小端序 (Little Endian) 编码，用于文件识别与快速定位。

偏移 字段名称 数据类型 长度 描述
0 Magic uint64 8字节 文件标识，固定为0x7152455354000000("qREST")
8 MetadataSize uint32 4字节 元数据部分长度（字节）
12 DataSize uint32 4字节 数据体部分长度（字节）

结构体定义 (C 语言风格):

typedef struct {
    uint64_t magic;           // 0x7152455354000000 ("qREST")
    uint32_t metadata_size;   // 元数据长度
    uint32_t data_size;       // 数据体长度
} QRestFileHeader;

文件头校验流程:

  1. 读取文件前 8 字节，检测 Magic 是否为0x7152455354000000("qREST")
  2. 读取 MetadataSize，定位元数据部分
  3. 读取 DataSize，定位数据体部分

3.元数据字段详细说明

建筑结构轻量化地震监测数据（Light-weight S2HM）的元数据（Metadata）是对监测数据本身的描述信息，包含了数据来源、采集环境、传感器配置、时间属性等关键信息。元数据是理解和正确使用监测数据的基础，确保数据的可追溯性、可比性和可重用性。

编码格式：

  • 字符编码: UTF-8
  • 格式: JSON，以二进制方式存储（无BOM，无末尾空字符）

3.2 建筑信息模块 (BuildingInfo)

建筑信息模块包含了监测项目的基本描述、地理位置信息、结构类型、结构轮廓以及各特征层的标高信息。该部分和监测无关，在建筑施工完成后基本保持不变。

字段名称 数据类型 描述 示例值
ProjectName String 监测项目或建筑物的唯一标识名/编号。 "Kunming_SSJY"
GeoLocation Object 建筑物的绝对地理位置信息。 见下文3.2.1
StructuralType String "结构类型描述（如 SteelFrame，RCFrame，ShearWall，Masonry 等）。" "SteelFrame"
StructuralFootprint Object 建筑物底平面的二维几何轮廓描述。 见下文3.2.2
ElevationNum Integer 结构在垂直方向上被划分的标高层数（包括地下室、地坪、各楼层和屋顶）。 16
Elevation Array(Float) 各特征楼层的绝对标高数组。负值代表地下层。 [-2.7, 0.0, ...]

3.2.1 地理信息

字段名称 数据类型 描述 示例值
Longitude Float 建筑物所在位置的经度坐标。 25.04
Latitude Float 建筑物所在位置的纬度坐标。 102.70
NorthAngle Float 建筑物 X 轴正向与地理正北方向的夹角，单位为度（°）。 0.0

3.2.2 结构轮廓

字段名称 数据类型 描述 示例值
Shape String 截面形状。支持Rectangular(矩形)、Circular(圆形)、Polygon(多边形)等。 "Rectangular"
Parameters Object 形状参数：根据不同形状提供相应的参数，见备注 {"Length": 41.2, "Width": 25.2}
BoundingBox Object 外接包围盒坐标。以几何中心为原点，标定结构在 XY 平面内的最大边界。 {"MaxX": 20.6, "MinX": -20.6, "MaxY": 12.6, "MinY": -12.6}
  • 备注：根据不同形状提供相应的参数：
    1. Rectangular 形状，Parameters 包含 Length 和 Width；
    2. Circular 形状，Parameters 包含 Radius；
    3. 对于 Polygon 形状，Parameters 包含一个顶点坐标数组：[[x1, y1], [x2, y2], ...]。

3.3 传感器布设模块 (InstrumentInfo)

传感器布设模块详细描述了监测系统中每个测量通道的物理属性、安装位置和测量参数。该部分在监测布设完成后基本保持不变，除非进行传感器维护或重新布设。

字段名称 数据类型 描述 示例值
Provider String 检测系统的提供者。 "SSJY"
ChannelNum Integer 18 当前数据块包含的有效测量通道总数。
Channels Array(Object) 见下文3.3.1 通道参数数组，每个对象代表一个独立的物理测量通道。

3.3.1 通道参数

字段名称 数据类型 描述 示例值
ChannelNo Integer 通道编号，从 1 开始递增。 1
ChannelID String 通道唯一标识符，通常和设备、通道编号相关联。 "SSJY_01"
Measurand String 测量物理量的类型（如 Acceleration、Velocity、Displacement）。 "Acceleration"
Scale Float 测量值的缩放因子，用于保证将数据体部分的波形数据转化为元数据中规定单位下的波形数据。 1.0
Azimuth Float 通道测量方向的方位角，单位为度（°）。以结构局部坐标系的 X 轴正向为 0°，逆时针旋转至 360°，竖直方向使用-1。 90.0
LocationXYZ Array(Float) 通道安装位置的三维坐标，以结构局部坐标系为参考。 [-20.6, -4.2, -2.7]
  • 备注：关于Scale字段的说明：
    Scale 字段定义了测量值的缩放因子，用于将数据体部分的原始波形数据转换为元数据中规定单位下的波形数据。具体来说，数据体中的时序数据通常以整数或浮点数形式存储，如使用Count值、ADC值和非文档规定单位的数值。为了确保这些数据能够正确地转换为元数据中规定的物理单位（如 m/s²、mm/s、mm 等），需要使用 Scale 字段中定义的缩放因子进行转换。举例计算方式如下：
    某数采输出物理量位电压值(mV)，灵敏度系数为 100mV/g，元数据中定义单位制位["m", "s"]，则 Scale 的计算方式为：
    Scale=1.0 m/s2100 mV/g×9.81 m/s2=0.0001019368 m/s2/mVScale = \frac{1.0 \, m/s^2}{100 \,mV/g \times 9.81 \, m/s^2} = 0.0001019368 \, m/s^2/mVScale=100mV/g×9.81m/s21.0m/s2​=0.0001019368m/s2/mV
    总的来说，Scale 字段的作用是确保数据体中的原始数值能够正确地转换为元数据中规定的物理单位，从而保证数据的一致性和可用性。正常使用时如果能直接得到物理单位下的数值，则 Scale 可以设置为 1.0；如果数据体中的数值需要经过缩放才能得到物理单位下的数值，则 Scale 应设置为相应的缩放因子。

3.4 时序数据信息模块 (DataInfo)

时序数据信息模块定义了当前数据块的时间属性和采样参数。该部分对于每个数据记录是确定的，必须严格遵守以确保数据的正确解析和时间对齐。

字段名称 数据类型 描述 示例值
EventName String 地震事件的唯一标识名/编号。 "2025_MYANMAR_7.9"
StartTime String 数据记录的起始时间，采用 ISO 8601 格式，包含时区信息。 "2025-03-28T14:20:00.000+08:00"
NPTS Integer 数据记录中每个通道的采样点数。 30000
DT Float 采样间隔，单位为秒。 0.02
Corrected String 数据是否或由谁进行了校正处理。可选值包括处理者名称，如"Machine"（表示机器自动处理）和"NULL"（表示未进行处理）等。 "NULL"

4.时序数据格式说明

时序数据部分采用二进制格式进行编码，以高效存储和传输大规模的监测数据。时序数据分为数据体包头和数据体包体两部分，包头包含了当前数据块的时间属性和采样参数，包体则包含了每个通道的时序数据，按照预定义的顺序进行存储，以便程序能够正确读取和处理。组织格式如下图：

4.1 数据体包头字段说明

包头固定为 32 字节，采用 小端序 (Little Endian) 编码：

偏移 字段名称 数据类型 长度 描述
0 Magic uint16 2 字节 包头标识，固定为 0x7144 ("qD")
2 SourceID uint16 2 字节 数据源 ID
4 Version uint8 1 字节 协议版本号，当前为 0x01
5 PacketType uint8 1 字节 数据包类型，见4.1.1
6 ChannelCount uint16 2 字节 通道数量
8 DataEncodings uint16 2 字节 数据编码方式，见4.1.2
10 SamplingRate uint16 2 字节 采样率 (Hz)
12 DataPointCount uint32 4 字节 每个通道的数据点数量
16 Timestamp uint64 8 字节 时间戳（毫秒，Unix epoch）
24 BodySize uint32 4 字节 数据包长度（字节）
28 Checksum uint32 4 字节 数据包 CRC32 校验和，见4.1.3
  • 备注：
    BodySize 字段指示了紧随包头之后的数据包部分的字节长度，接收端可以根据该字段值正确读取数据包内容。其计算方式为：
    BodySize=ChannelCount×DataPointCount×TypeSizeBodySize = ChannelCount × DataPointCount × TypeSizeBodySize=ChannelCount×DataPointCount×TypeSize
    其中 TypeSize 由 DataEncodings 字段指定。

结构体定义 (C 语言风格):

typedef struct {
    uint16_t magic;           // 0x7144 ("qD")
    uint16_t source_id;      // 数据源ID
    uint8_t  version;         // 协议版本 0x01
    uint8_t  packet_type;     // 数据包类型
    uint16_t channel_count;   // 通道数量
    uint16_t data_encodings; // 数据编码方式
    uint16_t sampling_rate;   // 采样率(Hz)
    uint32_t data_point_count;// 每个通道的数据点数量
    uint64_t timestamp;       // 时间戳（毫秒）
    uint32_t body_size;       // 数据体长度
    uint32_t checksum;        // CRC32校验和
} QRestPacketHeader;

4.1.1 数据包类型定义

PacketType 字段，其值固定为0x01。

4.1.2 数据编码方式定义

DataEncodings 字段定义了数据体中时序数据的编码方式，以确保发送端和接收端能够正确解析和处理数据。以下是 DataEncodings 的定义：

DataEncodings 值 编码方式 说明
0 32-bit 浮点数 (Float32) 每个数据点使用 4 字节表示，适用于大多数监测数据
1 64-bit 浮点数 (Float64) 每个数据点使用 8 字节表示，适用于需要高精度的场景
10 16-bit 整数 (Int16) 每个数据点使用 2 字节表示，适用于资源受限的场景
11 32-bit 整数 (Int32) 每个数据点使用 4 字节表示，适用于需要较高精度但又受限于存储空间的场景

4.1.3 校验和定义

Checksum 字段使用 CRC32 算法计算数据体部分的校验和，以确保数据在传输过程中未被篡改或损坏。发送端在生成数据包时会计算数据体的 CRC32 校验和，并将结果存储在 Checksum 字段中。接收端在接收到数据包后，首先根据 BodySize 字段读取数据体内容，然后使用相同的 CRC32 算法计算接收到的数据体的校验和，并与包头中的 Checksum 字段进行比较。如果两者匹配，则说明数据体未被篡改，可以安全使用；如果不匹配，则说明数据可能已损坏或被篡改，接收端应丢弃该数据包并记录错误日志。

算法参数：

参数 值
Polynomial 0x04C11DB7
Init 0xFFFFFFFF
RefIn False
RefOut False
XorOut 0xFFFFFFFF

算法代码示例：

uint32_t crc32(const uint8_t *data, size_t length) {
    uint32_t crc = 0xFFFFFFFF; 

    for (size_t i = 0; i < length; i++) {
        crc ^= data[i];

        for (int j = 0; j < 8; j++) {
            if (crc & 1) {
                crc = (crc >> 1) ^ 0xEDB88320;
            } else {
                crc >>= 1;
            }
        }
    }

    return ~crc;
}

4.2 数据体包体字段说明

数据体包体包含了每个通道的时序数据，按照预定义的顺序进行存储。每个通道的数据点数量由包头中的 DataPointCount 字段指定，数据编码方式由 DataEncodings 字段指定。接收端需要根据这些信息正确解析和处理数据体中的时序数据，以确保数据的正确使用和分析。
