qREST_DATA 结构监测数据头文件格式说明书

1.概述

本数据头文件基于 JSON 格式设计，旨在为土木工程结构健康监测（SHM）及强震动观测提供标准化的元数据（Metadata）存储规范。数据头包含三大核心模块：建筑结构信息、传感器布设信息以及时序数据属性。另外数据头文件还接受等价的 YAML 格式，以满足不同用户的使用习惯和工具兼容性需求。数据头文件的设计原则是清晰、简洁且具有高度可扩展性，能够适应未来监测项目中可能出现的新需求和新技术。

2.核心坐标与物理约定

在解析本数据文件时，需严格遵循以下工程约定：

  • 结构局部坐标系 (Local Coordinate System)：
    • 原点 (Origin)：以建筑底平面（正负零标高处）的几何中心为三维坐标系的原点 [0.0, 0.0, 0.0]。
    • 轴向 (Axes)：遵循右手笛卡尔坐标系。通常 X 轴与建筑主轴线（如长边）平行，Y 轴与次轴线（如短边）平行，Z 轴垂直向上。
  • 方向角定义 (Azimuth / Direction)：
    • 采用极坐标角度定义方式：以结构局部坐标系的 X 轴正方向为 0°，在 XY 平面内逆时针旋转至 360°。例如：Y 轴正向为 90°，X 轴负向为 180°。
  • 地理正北角 (North Angle)：
    • 以结构局部坐标系的 X 轴正方向与地理正北方向之间的夹角定义，单位为度 (°)。例如：如果 X 轴正向与地理正北方向重合，则 North Angle 为 0°；如果 X 轴正向指向东，则 North Angle 为 90°。

字段详细说明

头文件字段按照模块化结构进行组织，包含以下主要部分：

3.1 根节点 (Root Nodes)

字段名称 数据类型 示例值 描述
Header String 数据头信息，文件标识符，用于程序快速识别文件格式。 "qREST_DATA"
Version Array(Int) 数据协议版本号，采用[主版本,次版本,修订号]的形式。 [1, 0, 0]
BuildingInformation Object 监测对象（建筑结构）的详细物理与几何参数集。 见下文3.2
MeasurementInformation Object 传感器布设与测量配置参数。 见下文3.3
DataInformation Object 时序数据属性。 见下文3.4

3.2 建筑信息模块 (BuildingInformation)

建筑信息模块包含了监测项目的基本描述、地理位置信息、结构类型、结构轮廓以及各特征层的标高信息。该部分和监测无关，在建筑施工完成后基本保持不变。

字段名称 数据类型 描述 示例值
ProjectName String 监测项目或建筑物的唯一标识名/编号。 "Kunming_SSJY"
GeoLocation Object 建筑物的绝对地理位置信息。 见下文3.2.1
StructuralType String "结构类型描述（如 SteelFrame，RCFrame，ShearWall，Masonry 等）。" "SteelFrame"
StructuralFootprint Object 建筑物底平面的二维几何轮廓描述。 见下文3.2.2
ElevationNum Integer 结构在垂直方向上被划分的标高层数（包括地下室、地坪、各楼层和屋顶）。 16
StructuralElevation Array(Float) 各特征楼层的绝对标高数组。负值代表地下层。 [-2.7, 0.0, ...]

3.2.1 地理信息

字段名称 数据类型 描述 示例值
Longitude Float 建筑物所在位置的经度坐标。 25.04
Latitude Float 建筑物所在位置的纬度坐标。 102.70
NorthAngle Float 建筑物 X 轴正向与地理正北方向的夹角，单位为度 (°)。 0.0

3.2.2 结构轮廓

字段名称 数据类型 描述 示例值
Shape String 截面形状。支持Rectangular(矩形)、Circular(圆形)、Polygon(多边形)等。 "Rectangular"
Parameters Object 形状参数：根据不同形状提供相应的参数，见备注 {"Length": 41.2, "Width": 25.2}
BoundingBox Object 外接包围盒坐标。以几何中心为原点，标定结构在 XY 平面内的最大边界。 {"MaxX": 20.6, "MinX": -20.6, "MaxY": 12.6, "MinY": -12.6}
Unit String 几何尺寸的物理单位（通常为米）。 "m"
  • 备注：根据不同形状提供相应的参数：
    1. Rectangular 形状，Parameters 包含 Length 和 Width；
    2. Circular 形状，Parameters 包含 Radius；
    3. 对于 Polygon 形状，Parameters 包含一个顶点坐标数组：[[x1, y1], [x2, y2], ...]。

3.3 传感器布设模块 (MeasurementInformation)

传感器布设模块详细描述了监测系统中每个测量通道的物理属性、安装位置和测量参数。该部分在监测布设完成后基本保持不变，除非进行传感器维护或重新布设。

字段名称 数据类型 描述 示例值
ChannelNum Integer 18 当前数据块包含的有效测量通道总数。
Channels Array(Object) 见下文3.3.1 通道参数数组，每个对象代表一个独立的物理测量通道。

3.3.1 通道参数

字段名称 数据类型 描述 示例值
ChannelID Integer 1 通道的唯一序号（从 1 开始）。
SerialNumber String "UNKNOWN" 传感器的硬件序列号。
Measurand String "Acceleration" 测量的物理量类型（如 Acceleration, Velocity, Displacement）。
Azimuth Float 90.0 传感器敏感轴的方向角。90.0 代表沿 Y 轴正向测量，0.0 代表沿 X 轴正向测量。
LocationXYZ Array(Float) [-20.6, -4.2, -2.7] 传感器的三维空间安装坐标 [X, Y, Z]。Z 值必须与 StructuralElevation 中的某个层高对应。
Unit String "m/s2" 当前通道输出数据的物理单位。

3.4 时序数据信息模块 (DataInformation)

时序数据信息模块定义了当前数据块的时间属性和采样参数。该部分对于每个数据记录是确定的，必须严格遵守以确保数据的正确解析和时间对齐。

字段名称 数据类型 描述 示例值
EventID String "2025_MYANMAR_7.9" 触发本次数据记录的事件唯一标识（如特定地震编号、台风编号或日常时间戳）。
StartTime String "2025-03-28T14:20:00.000+08:00" 数据段的首个采样点发生时间。严格遵循 ISO 8601 标准并携带时区（如 +08:00 为北京时间）。
NPTS Integer 30000 Number of Points，该段数据每个通道包含的采样点总数
DT Float 0.02 Delta Time，相邻两个采样点之间的时间间隔（秒）。由此可推导采样率为 1 / DT。
